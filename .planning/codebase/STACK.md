## Languages

**Primary:**
- C++20 - Core algorithm implementations in `cuml-main/cpp/src/` and `cuml-main/cpp/include/`
- CUDA - GPU kernels and device code, compiled by nvcc, co-located with C++ in `cuml-main/cpp/src/` (`.cu` files)
- Python 3 (>=3.11) - High-level API, bindings, and the `cuml.accel` drop-in accelerator layer

**Secondary:**
- Cython (>=3.2.2) - FFI bridge between Python and libcuml C++ shared library; `.pyx` and `.pxd` files throughout `cuml-main/python/cuml/cuml/`
- CMake (>=4.0) - Build system for the C++/CUDA layer

## Frameworks

**Core C++/CUDA:**
- CMake >=4.0 — project build system (`cuml-main/cpp/CMakeLists.txt`, `cuml-main/python/libcuml/CMakeLists.txt`, `cuml-main/python/cuml/CMakeLists.txt`)
- rapids-cmake — RAPIDS CMake helpers (fetched via `cuml-main/cmake/RAPIDS.cmake`; modules in `cuml-main/cmake/rapids_config.cmake`)
- CPM (CMake Package Manager) — third-party dependency fetching, used throughout `cuml-main/cpp/cmake/thirdparty/`
- Ninja — preferred build generator (required; `ninja.make-fallback = false` in pyproject.toml)
- OpenMP — CPU thread parallelism; optional, detected at configure time (`DISABLE_OPENMP` option)

**Python Build:**
- scikit-build-core >=0.11.0 — PEP 517 build backend that drives CMake from Python (`build-backend = "scikit_build_core.build"`)
- rapids-build-backend >=0.4.0,<0.5.0 — RAPIDS wrapper around scikit-build-core; injects RAPIDS-specific dependency matrix logic
- Cython >=3.2.2 — compiles `.pyx` files into C extensions

**Python Runtime:**
- numpy >=1.26,<3.0
- scipy >=1.14.0
- numba >=0.60.0,<0.65.0 — JIT-compiled GPU kernels from Python
- numba-cuda >=0.22.2,<0.29.0 — CUDA backend for numba
- cupy >=13.6.0 (!=14.0.0, !=14.1.0) — GPU array library (NumPy-compatible)
- joblib >=0.11 — parallelism utilities
- scikit-learn >=1.6 — API compatibility baseline + estimator interface
- treelite >=4.7.0,<5.0.0 — model serialization for tree-based models
- rich — terminal output formatting
- packaging — version parsing utilities

**Testing:**
- pytest — test runner; config in `cuml-main/python/cuml/pyproject.toml` (`[tool.pytest.ini_options]`)
- pytest-benchmark — micro-benchmarks
- pytest-cases — parametrized test cases
- pytest-cov — coverage
- pytest-xdist — parallel test execution
- Google Test (GTest) — C++ unit tests, fetched via CPM: `rapids_cpm_gtest(BUILD_STATIC)`
- Google Benchmark (GBench) — C++ benchmarks, fetched via CPM: `rapids_cpm_gbench(BUILD_STATIC)`
- hypothesis >=6.0,<7 — property-based testing
- dask-ml >=2024 — distributed ML testing

**Documentation:**
- Doxygen 1.9.1 — C++ API docs (`cuml-main/cpp/Doxyfile.in`)
- Sphinx — Python API docs (`cuml-main/docs/`)
- numpydoc, nbsphinx, recommonmark, sphinx-copybutton, sphinx-markdown-tables

## Configuration

**Environment / Build Matrix:**
- CUDA versions: 12.9, 13.2 (primary CI); also 12.2, 12.5, 12.8, 13.0, 13.1 supported
- Python versions: 3.11, 3.12, 3.13, 3.14
- Architectures: x86_64, aarch64
- All matrix combinations managed in `cuml-main/dependencies.yaml` via `rapids-dependency-file-generator`
- Generated conda environment files: `cuml-main/conda/environments/*.yaml`

**CMake Build Options (key flags in `cuml-main/cpp/CMakeLists.txt`):**

| Option | Default | Purpose |
|--------|---------|---------|
| `CUML_ENABLE_GPU` | ON | Enable GPU-accelerated algorithms |
| `BUILD_CUML_CPP_LIBRARY` | ON | Build libcuml shared library |
| `BUILD_CUML_TESTS` | ON | Build C++ algorithm tests |
| `BUILD_PRIMS_TESTS` | ON | Build ML primitives tests |
| `BUILD_CUML_BENCH` | ON | Build C++ benchmark suite |
| `BUILD_CUML_MPI_COMMS` | OFF | Build MPI+NCCL communicator |
| `SINGLEGPU` | OFF | Disable multi-GPU (mnmg) components |
| `CUML_ALGORITHMS` | "ALL" | Selectively build algorithm subsets |
| `DISABLE_OPENMP` | OFF | Disable OpenMP |
| `NVTX` | OFF | Enable NVTX profiling markers |
| `USE_CCACHE` | OFF | Enable ccache for build caching |
| `CUML_USE_CUVS_STATIC` | OFF | Statically link cuVS |
| `CUML_USE_TREELITE_STATIC` | OFF | Statically link Treelite |
| `CUDA_STATIC_MATH_LIBRARIES` | OFF | Statically link CUDA math libs |

**Algorithm-level algorithm grouping** (`cuml-main/cpp/cmake/modules/ConfigureAlgorithms.cmake`):
- `cluster` → dbscan, hdbscan, kmeans, hierarchicalclustering, spectralclustering
- `decomposition` → pca, tsvd
- `ensemble` → randomforest
- `linear_model` → linearregression, ridge, lasso, logisticregression, solvers
- `manifold` → tsne, umap
- `tsa` → arima, autoarima, holtwinters

**Python build configuration:**
- `cuml-main/python/cuml/pyproject.toml` — cuml Python wheel config
- `cuml-main/python/libcuml/pyproject.toml` — libcuml C++ wheel config (cmake prefix entry-point)
- `[tool.scikit-build]` sections configure CMake build type (Release), build dir, wheel contents

**Compiler flags:**
- CUDA: `--expt-extended-lambda --expt-relaxed-constexpr -Werror=all-warnings`
- C++: `-Wall -Werror -Wno-unknown-pragmas`
- Fatbin compression enabled via `rapids_cuda_enable_fatbin_compression`
- Defined in `cuml-main/cpp/cmake/modules/ConfigureCUDA.cmake`
