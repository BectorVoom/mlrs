<objective>
Deliver `Ridge` (LINEAR-02) — the Cholesky normal-equations solver (D-02),
consuming the Cholesky primitive validated standalone in 04-02. Solve
`(XᵀX + αI)·coef = Xᵀy` via the raw Gram (`gemm(transa=true)`, RESEARCH Open Q1)
+ diagonal-α + `prims::cholesky`, with the intercept recovered via center-then-
solve (D-05, α never penalizes the intercept). Then extend the build-failing
PoolStats memory gate (D-03) to the estimator fit→predict/transform pipelines:
device-residency, bounded pool reuse across repeated same-shape calls, and the
Ridge Gram-buffer reuse for the Cholesky factor.

Ridge is sequenced LAST among the linear models because it depends on the
Cholesky primitive (04-02) and shares `linear/mod.rs` with LinearRegression
(04-03). It deliberately uses a DIFFERENT solver from LinearRegression (Cholesky
vs SVD) — do not unify (D-02).

Purpose: complete the linear-model surface with the SPD solver and prove the
whole phase composes on-device under the memory gate.
Output: `Ridge` struct; activated `ridge_test.rs`; extended `memory_gate_test.rs`.
</objective>
