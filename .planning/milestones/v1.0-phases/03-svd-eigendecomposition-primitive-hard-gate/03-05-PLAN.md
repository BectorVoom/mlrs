<objective>
Extend the Phase-2 build-failing PoolStats memory gate to the iterative SVD/eig primitive
with the three HARD D-11 assertions: (1) bounded Jacobi scratch (allocations don't grow
with sweep count), (2) eig reuses the covariance/GEMM output buffer, (3) no host round-trip
between sweeps (read_backs == 1). This is the phase's memory-efficiency gate — PROJECT.md
requires memory efficiency verified per phase, not deferred — and it is a blocking
acceptance criterion, not optional.

Purpose: the iterative sweep is the first multi-pass device loop in the project; without
these gates a per-sweep allocation or a mid-sweep read-back could regress silently. The
gates assert on the existing pool counters, exactly as the Phase-2 D-10 gate does.
Output: three new green, build-failing memory-gate assertions in memory_gate_test.rs.
</objective>
