<objective>
Land the NEW DBSCAN eps-region + core-mask primitive (D-04): the DEVICE computes the n² pairwise squared-distance matrix (Phase-2 distance prim), thresholds it at `eps²` (`<= eps` on Euclidean = `<= eps²` on squared), counts each point's self-inclusive eps-neighbor count, and marks it core when that count ≥ min_samples. The wrapper reads the core mask + the eps-adjacency back to host as the single documented round-trip — the host DFS expansion is the ESTIMATOR's job (plan 07), NOT this prim (D-04: inherently sequential graph traversal is host-side). Lands with its standalone oracle BEFORE the DBSCAN estimator consumes it.

Purpose: CLUSTER-02 device-compute half. The n² distance matrix is the accepted brute-force memory cost (documented memory-gate exception, plan 11).
Output: `dbscan.rs` kernel + `prims/dbscan.rs` (eps_core_mask) + `dbscan_mask_test.rs`.
</objective>
