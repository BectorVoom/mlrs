Create `crates/mlrs-backend/tests/memory_gate_test.rs` with three build-failing
tests asserting on `pool.stats()`:

1. `memory_gate_reuse_bounded`: thread ONE `BufferPool` and run a primitive (e.g.
   `distance`) N times (N≈5) at the SAME shape, reusing the caller-provided
   out-buffer + pool scratch (D-11). Capture `FIRST_ITER_ALLOCS = stats.allocations`
   after iteration 1, then assert after iterations 2..N that `stats.allocations <=
   FIRST_ITER_ALLOCS` (bounded, NOT ∝N) AND `stats.reuses >= N - 1` (reuse > 0).

2. `memory_gate_no_midpipeline_readback`: build a chained GEMM→reduce→distance
   pipeline entirely `DeviceArray`→`DeviceArray` (no `to_host` between stages), then
   a SINGLE terminal metered read-back for the compare. Assert
   `stats.read_backs == 1` (the only read-back is the terminal one — proves zero
   mid-pipeline host round-trips, D-10 gate 2). Uses the Plan-01 `read_backs`
   counter / metered read-back path.

3. `memory_gate_gram_reuses_gemm_buffer`: run a `gemm` producing an
   `n_features × n_features` output, then call `covariance` passing that GEMM
   output `DeviceArray` as covariance's `out` (D-11). Assert that computing the
   covariance does NOT raise `stats.allocations` beyond the GEMM's own (the Gram
   output reuses the GEMM handle / a free-list buffer of the same byte size →
   `reuses` bumps, `allocations` does not) — D-10 gate 3.

Each test is a plain `#[test]` (no in-source mod tests, AGENTS.md §2), logs the
backend line, and runs on cpu (the counter assertions are backend-agnostic; also
green on wgpu).
