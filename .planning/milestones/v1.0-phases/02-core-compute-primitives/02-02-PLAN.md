<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host slice → device buffer | inputs uploaded via `DeviceArray::from_host`; `len` is the validated read-back source of truth (T-04-01) |
| caller `(rows, cols)` axis geometry → kernel grid | row/column reductions derive grid dims from caller geometry; must satisfy `rows*cols == len` (D-04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0202-01 | Tampering | `ArrayArg::from_raw_parts(handle, len)` in reduction launches | mitigate | `len` from validated `DeviceArray.len`; every kernel bounds-checks against `input.len()` (T-04-01 carried forward) |
| T-0202-02 | Tampering | caller axis geometry `(rows, cols)` vs `len` | mitigate | assert `rows*cols == len` at the host-API entry before launch (D-04, ASVS V5) |
| T-0202-03 | Denial of Service | plane path on adapters lacking subgroup support | accept | Skip-with-log (never fail) — the shared-memory path is the guaranteed-portable fallback; a missing adapter feature degrades to the fallback, not a crash |
| T-0202-SC | Tampering | cargo dependency | mitigate | Zero new dependencies — pure `#[cube]` kernels (RESEARCH §Don't Hand-Roll); nothing installed |

No high-severity threats: numerical compute-kernel phase, no auth/session/network/PII surface.
</threat_model>
