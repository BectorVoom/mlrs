<objective>
Extend `mlrs_core::label_perm` with a `-1`-pinned label-permutation matcher
(`best_match_accuracy_pinned_noise`) for the HDBSCAN exact-label gate (HDBS-02).
The noise sentinel `-1` must map ONLY to `-1` (and `-1` must map only from `-1`);
all non-noise labels match up to permutation via the existing greedy matcher.

Purpose: HDBSCAN labels are a permutation of sklearn's EXCEPT the `-1` noise
class, which is a fixed point — a point sklearn calls noise that mlrs calls a
cluster member (or vice-versa) is a genuine failure, never a relabeling. The
existing `best_match_accuracy` would wrongly permute `-1` into a cluster id.
Output: the new matcher + its `pub use` + a unit test, all in `mlrs-core`.

Output: `best_match_accuracy_pinned_noise` reachable as
`mlrs_core::best_match_accuracy_pinned_noise`.
</objective>
