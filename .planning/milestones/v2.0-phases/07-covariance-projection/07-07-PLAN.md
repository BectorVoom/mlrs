<objective>
Wrap the five Phase-7 estimators as PyO3 `#[pyclass]` objects on `_mlrs` and add their
pure-Python sklearn-compatible shims, reusing the SHIPPED `any_estimator!` machinery —
v2 adds ZERO binding infrastructure (RESEARCH). This introduces the first v2 `partial_fit`
method (IncrementalPCA) and the `johnson_lindenstrauss_min_dim` `#[pyfunction]`. Each
device-compute body honors the two load-bearing contracts: GIL release via `py.detach`
(PY-03) and `guard_f64()?` BEFORE the F64 arm (D-04); accessors are dtype-suffixed.

Purpose: the user-facing completion of COV-01/COV-02/DECOMP-03/PROJ-01/PROJ-02 — a data
scientist can `import mlrs` and fit/partial_fit/transform these estimators. This is the
incremental per-phase Python wrapping the ROADMAP assigns (PY-06 final sign-off remains
Phase 11; this plan does NOT claim PY-06).
Output: covariance.rs + projection.rs PyO3 wrappers, IncrementalPCA partial_fit method,
johnson_lindenstrauss_min_dim pyfunction, four Python shim modules, smoke test extended.
</objective>
