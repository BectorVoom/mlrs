## Decisions Made
- **Shared-dimension guard in `kernel_matrix` (Rule 2):** added a `PrimError::DimMismatch` on `cols == cols_y` beyond the plan's enumerated checks — `K(X, Y)` is only defined when X and Y live in the same feature space; without it a mismatched-feature call would reach the base-op dispatch and produce an out-of-bounds device read.
- **`Kernel<F>` real `degree`:** `Poly { degree: F }` stores the polynomial degree as a real `F` (not an integer) so the Wave-1 map can use `F::powf` — the sklearn-faithful real-exponent form (RESEARCH Pitfall 7).
- **Single-fixture multi-path coverage:** the kernel_matrix/kernel_ridge fixtures store both a resolved-default-gamma and an explicit-gamma case (D-05); the kernel_density fixture stores sklearn's resolved `bandwidth_` so the Wave-2 host closed form (D-09) is pinned directly.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Added a shared-feature-dimension guard to `kernel_matrix::validate_geometry`**
- **Found during:** Task 2 (kernel_matrix.rs stub)
- **Issue:** The plan listed `rows_x*cols == x.len()`, `rows_y*cols == y.len()`, empty-geometry, and `out` length checks, but not an explicit check that X and Y share the feature dimension. `K(X, Y)` is undefined when `cols != cols_y`, and an unchecked mismatch would flow into the Wave-1 GEMM/distance base op as an out-of-bounds read (T-08-01-01 / ASVS V5 validate-before-launch).
- **Fix:** Added a `PrimError::DimMismatch { dim: "n_features", lhs: cols, rhs: cols_y }` return when `cols != cols_y`, before the empty-geometry and out-length checks.
- **Files modified:** crates/mlrs-backend/src/prims/kernel_matrix.rs
- **Verification:** `cargo build -p mlrs-backend --features cpu` succeeds; the guard is a typed early-return, consistent with the covariance.rs / gemm.rs validate-before-launch precedent.
- **Committed in:** `0f90f0c` (Task 2 commit)
