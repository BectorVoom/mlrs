## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python caller → _mlrs #[pyclass] | Untrusted sklearn-named strings + scalars + array shapes cross the FFI here |
| fit body TryFrom + build() | enum strings + data-independent params validated → ValueError |
| py.detach → device | array buffers + dtype reach the validated estimator/prim |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05-01 | Tampering | enum string + BuildError at FFI | mitigate | Unknown loss/penalty/lr string and bad data-independent params → typed BuildError → PyValueError via build_err_to_py (D-05/D-09); sklearn-faithful, never a panic. |
| T-10-05-02 | Denial of Service | panic across the PyO3 boundary on device launch failure | mitigate | prim/fit errors captured into AlgoError → algo_err_to_py PyValueError; py.detach + lock_pool() (poison-recovering) prevent an unwinding panic across FFI. |
| T-10-05-03 | Information Disclosure | f64 on an f64-incapable backend | mitigate | crate::capability::guard_f64() gates the F64 arm BEFORE upload (rocm has no f64); returns a typed error, not a device fault. |
| T-10-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs (v2 Cargo-unchanged; pyo3 stays 0.28; RESEARCH §Package Legitimacy Audit: none). |

No high-severity threats: every untrusted FFI input (enum strings, hyperparameters, shapes, dtype) is funneled into a typed PyValueError before any device work; the f64 guard prevents an unsupported-backend fault.
