# Phase 15 Plan 06: GLOSH outlier_scores_ + store_centers Summary

GLOSH `outlier_scores_` (HDBS-03) now match the `hdbscan` 0.8.44 library at 0.0 diff via a
parallel hdbscan-convention condensed tree, and `store_centers` (HDBS-04) produces
`centroids_`/`medoids_` matching sklearn within 1e-5 under the same label permutation —
completing the full HDBSCAN surface under every value gate.

## What was built

### Task 1 — GLOSH outlier_scores_ via the hdbscan-convention tree (HDBS-03), commit `3c07dd9`

The prior executor escalated a D-06 decision: mlrs's sklearn-matching tree diverged ~0.06 from
the committed hdbscan-0.8.44 `outlier_scores` oracle. **Root cause (verified in a `/tmp` venv):**
the oracle is generated by `hdbscan.HDBSCAN(..., algorithm='generic')`, whose tree differs from
the sklearn `labels_`/`probabilities_` pipeline in TWO ways, BOTH required to reproduce it:

1. **Core distance at index `min_samples`** (not `min_samples-1`). hdbscan's
   `mutual_reachability` uses `np.partition(D, min_points, axis=0)[min_points]`.
2. **hdbscan's `mst_linkage_core` tie-order** — a dense Prim tracking per-node
   `current_distances`/`current_sources` with non-strict `d < current_distances[j]` update and
   strict `current_distances[j] < new_distance` selection — distinct from the sklearn `argmin` Prim.

**User decision: Option A** — build a GLOSH-only parallel tree with hdbscan's exact convention.
`glosh::hdbscan_outlier_scores(dist, n, min_samples, min_cluster_size)` does:
core distance at index `min_samples` → mutual reachability → `mst_linkage_core` (ported verbatim)
→ argsort → single linkage → `condense_tree(min_cluster_size)` → the existing GLOSH
`outlier_scores` pass. `labels_`/`probabilities_` are **untouched** (still sklearn-exact; all 34
prior gates pass). `fit` builds the alpha-scaled dense distance matrix (precomputed: `X`;
feature metrics: rebuilt host-side via `feature_metric_dense_distances`) and feeds GLOSH.

`outlier_scores_match_{f32,f64}` pass at 0.0 diff vs the committed fixture.

### Task 2 — store_centers → centroids_/medoids_ (HDBS-04), commit `b2af946`

`centers::weighted_cluster_center` ports sklearn `_weighted_cluster_center` verbatim:
centroid = `np.average(data, weights=strength)` (probability-weighted mean); medoid =
`data[argmin((pairwise_distances(data)*strength).sum(axis=1))]` (the distance matrix is weighted
by `strength` as a row vector, summed per row, argmin'd). Centers iterate ascending cluster id
`0..n_clusters`. Added `centroids_`/`medoids_` fitted fields + `centroids()`/`medoids()`
accessors; `fit` computes them when `store_centers` is `Some`. `store_centers` with
`Metric::Precomputed` errors with `AlgoError::Unsupported` BEFORE any compute (feature-array
only, sklearn parity / threat T-15-06-V5).

`centers_match_{f32,f64}` pass ≤1e-5 vs sklearn under the `best_mapping` permutation;
`store_centers_precomputed_errors` confirms the typed rejection.

GLOSH instead runs over a parallel hdbscan-convention tree. This is the user-approved resolution,
not an autonomous deviation. No other deviations — Task 2 followed the plan exactly.
