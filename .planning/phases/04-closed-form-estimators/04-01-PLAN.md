<objective>
Wave-0 scaffold for Phase 4. Lay down everything the estimator and primitive
plans build against so they compile and have failing tests on day one: the
`mlrs-algos` dependency graph, the shared `Fit`/`Predict`/`Transform` trait
surface (D-04), the new `PrimError::NotPositiveDefinite` variant and an
estimator-side hyperparameter error, the `linear/` and `decomposition/` module
stubs (so 04-03/04/05 edit only their own files — `lib.rs` is owned here), the
`gen_oracle.py` fixture generators + committed `.npz` blobs for all five new
test targets, and the five test files as compiling `#[ignore]` stubs (Nyquist:
tests exist before implementation).

Purpose: every downstream Phase-4 plan (Cholesky primitive, LinearRegression,
PCA/TruncatedSVD, Ridge) consumes contracts created here. Defining them once
up front prevents executors from scavenging the codebase for trait shapes,
error variants, and fixture layouts, and keeps the estimator plans
file-disjoint so they run in parallel.

Output: `mlrs-algos/Cargo.toml` deps; `traits.rs`; `linear/mod.rs` +
`decomposition/mod.rs` stubs; `lib.rs` module index; `error.rs` new variants;
`gen_oracle.py` generators + committed fixtures; five `#[ignore]` test stubs.
</objective>
