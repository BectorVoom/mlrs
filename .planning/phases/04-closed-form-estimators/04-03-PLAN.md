<objective>
Deliver `LinearRegression` (LINEAR-01) — OLS via the SVD pseudo-inverse, matching
sklearn's default `scipy.linalg.lstsq` (gelsd) path. Compose the validated
Phase-3 thin SVD + Phase-2 column-mean centering + GEMM products on-device:
`coef = V·diag(σ⁺)·Uᵀ·y_centered` with the small-singular-value cutoff
`σ⁺_i = 1/σ_i if σ_i > cond·σ_max else 0` (RESEARCH Pitfall 1). Intercept via
center-then-solve (D-05).

This is one of three SVD-backed estimators that build in parallel (no convergence
risk — the iterative risk was absorbed by Phase 3). It deliberately uses a
DIFFERENT solver from Ridge (SVD vs Cholesky) — do not unify them (D-02). Edits
only `linear/` files (04-01 owns `lib.rs`), so it runs parallel with PCA/TSVD.

Purpose: exercise the full Arrow→kernel→device-state→materialize→oracle pipeline
for the OLS path with sklearn-faithful rank-deficient handling.
Output: `LinearRegression` struct implementing `Fit`+`Predict`; activated
`linear_regression_test.rs`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host caller → LinearRegression::fit/predict | untrusted `(rows, cols)` geometry + `fit_intercept` cross into prim launches |
| singular values → σ⁺ reciprocal | a ~0 singular value (collinear X) risks division blow-up |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | DoS (inf/NaN) | σ⁺ reciprocal on tiny singular values | mitigate | Cutoff `σ⁺=0` below `cond·σ_max` (with NEAR_ZERO_FLOOR fallback) — no 1/0 blow-up (RESEARCH Pitfall 1) |
| T-04-03-02 | Tampering / DoS | fit/predict geometry | mitigate | Validate `n_samples>0`, `n_features>0`, X/y/coef shape consistency before launch (AlgoError, ASVS V5); the consumed prims (svd/gemm/reduce) re-validate their own geometry |
| T-04-03-03 | DoS (launch reject) | f64 launch on rocm/HIP | mitigate | `skip_f64_with_log` (D-07) — f64 skips on rocm, runs on cpu |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | No new cargo packages; sklearn fixtures are committed blobs from a /tmp venv, never shipped |
</threat_model>
