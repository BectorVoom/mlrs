<objective>
Build the two security- and portability-critical pieces of the device layer: the Arrow zero-copy bridge with HARD-REJECT validation (D-06) and the f64 capability gate (FOUND-04). The bridge is the single ingress for data into device buffers and the primary threat surface of this phase — untrusted/malformed Arrow buffers (offset/slice, set null bits, misalignment) must return a typed `BridgeError` BEFORE any `unsafe` transmute. The capability layer reports f64 support so f32 stays the portable wgpu baseline and f64 tests skip-with-log on adapters lacking SHADER_F64.

Purpose: ROADMAP Criterion 3 (bridge rejects sliced/nullable/misaligned before unsafe) and Criterion 4 (capability reports f64; f64 tests skip/xfail with logged reason; CI log shows dtype×backend). Maps to ASVS V5 input validation.
Output: `mlrs-backend::bridge` (validate_f32/validate_f64), completed `mlrs-backend::capability` (skip/xfail + logging), negative tests in `crates/mlrs-backend/tests/`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Arrow buffer (untrusted/malformed) → unsafe transmute → device | PRIMARY surface. Sliced/offset, nullable, or misaligned Arrow buffers reaching `bytemuck`/`unsafe` cause OOB read, aliased-parent-data upload, or silent wrong results. |
| wgpu adapter capability → f64 kernel launch | An f64 kernel launched on an adapter lacking SHADER_F64 yields runtime compile failure or garbage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / DoS | Unsafe transmute of misaligned/short Arrow buffer → UB / OOB read | mitigate | `bytemuck::try_cast_slice` (alignment+size checked) + explicit reject before any unsafe; `// SAFETY:` comment on every unsafe (ASVS V5). |
| T-03-02 | Information disclosure / Tampering | Sliced/offset Arrow array uploading aliased parent-buffer data | mitigate | Reject `offset() != 0` before transmute; negative test with `arr.slice(...)` asserts Err (Criterion 3). |
| T-03-03 | Tampering (silent wrong results) | Nullable array uploading meaningless null-slot values | mitigate | Reject `null_count() != 0`/`nulls().is_some()`; negative test with a None-containing array asserts Err (Criterion 3). |
| T-03-04 | Tampering | f64 oracle silently passing/failing on wgpu without SHADER_F64 | mitigate | Gate every f64 path on `supports_f64`; skip-with-log; always log dtype×backend (Criterion 4). |
| T-03-SC | Tampering | (no new package installs in this plan) | accept | All crates already audited; logger crate verified in Plan 02 checkpoint. |
