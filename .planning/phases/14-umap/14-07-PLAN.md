<objective>
Close GAP 1 — collapse CR-01 (BLOCKER) and CR-03 (WARNING) into ONE fix per REVIEW CR-01 "option b".

CR-01: The fit-path `host_epoch_driver` launches one cube per owner (`CubeCount::Static(n,1,1)`, umap.rs:1265) with `move_other=1` (umap.rs:1293). Under `move_other==1` each owner-cube also writes its positive neighbour's coordinates (`embedding[other_base+d1]`, umap_layout.rs:155-157). Because the fuzzy graph is symmetric, two concurrently-scheduled cubes read-modify-write the same vertex slots with NO synchronization → an unsynchronized data race on any genuinely parallel backend (wgpu/rocm/cuda). rocm is a REAL runnable parallel GPU gate (f32) per project memory, so the race is reachable. This breaks the D-05 byte-identical contract that criterion 3 and the project core value ("any CubeCL backend") require; it passes today only because the sole gate is the sequential cpu-MLIR backend.

CR-03: The symmetric COO contains both (r,c) and (c,r); `host_epoch_driver` builds one positive edge per COO entry keyed by owner=head (umap.rs:1190-1227), so each undirected pair yields two owner-edges, and with `move_other=1` each edge moves BOTH endpoints → every undirected pair is attracted ~2-4× per due-epoch vs umap-learn's single head/tail pass. `PROPERTY_EPS=0.02` was calibrated against this doubled output, so the gate validates against mlrs's own divergence.

Fix (REVIEW CR-01 option b): set the fit launch `move_other=0` over the already-symmetric COO. Each owner-cube then writes ONLY its own vertex (no foreign-vertex write → race eliminated), AND each undirected pair is processed once per direction (the symmetric graph already carries both (r,c) and (c,r)), matching umap-learn's single-pass schedule (double-count eliminated). One change resolves both findings.

Purpose: Make the stochastic SGD layout determinism contract (D-05) hold on ANY backend and the force schedule a faithful umap-learn port (criterion 3, UMAP-03), not a gate calibrated to a bug.
Output: `move_other=0` fit launch routed through a single `FIT_MOVE_OTHER` constant; an EXECUTABLE invariant test (`fit_move_other_is_zero`) that fails if `move_other=1` is restored (flag-equals-0 + slot-disjoint write-range partition) — encoding the D-05 cross-cube write guarantee, not a re-run of `reproducible_f64`; recalibrated PROPERTY_EPS/ARI_BAND under a hard ε ≤ 0.04 guardrail with a per-pair sample-count assertion; re-confirmed reproducible/property gates.
</objective>
