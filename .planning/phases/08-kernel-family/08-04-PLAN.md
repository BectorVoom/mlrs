    - gaussian(input=sqdist, h): `F::exp(-0.5 * sqdist / (h*h))` (squared dist).
    - epanechnikov(input=sqdist, h): `1 − sqdist/(h*h)` inside, 0 outside (compact; squared).
    - tophat(input=dist, h): `1` if `dist < h` else `0` (raw dist).
    - exponential(input=dist, h): `F::exp(-dist/h)` (raw dist, no compact support).
    - linear(input=dist, h): `1 − dist/h` inside, 0 outside (raw dist).
    - cosine(input=dist, h): `F::cos(0.5·π·dist/h)` inside, 0 outside (raw dist).
    The compact-support guard is STATEMENT-form (copy `clamp_nonneg` elementwise.rs:38-48):
    `let mut val = …; let zero = F::from_int(0); if d >= h { val = zero; } output[tid] = val;` —
    NEVER an if-expression, NEVER `F::INFINITY` (Pitfall 3). gaussian/epanechnikov consume
    `distance(sqrt=false)` (squared); tophat/exponential/linear/cosine consume `distance(sqrt=true)`
    (raw) — Pitfall 4. Transcendentals via static assoc fns only (`F::exp`/`F::cos`).

    elementwise `scale`-by-reciprocal (divide each row by its max for the optional rescale) — reuse
