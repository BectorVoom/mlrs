<objective>
Author the new `mlrs_algos::typestate` module — the sealed `State` marker trait, the `Unfit`/`Fitted` zero-sized marker types, and the four typestate-aware traits (`Fit`/`Predict`/`Transform`/`PartialFit`) that consume `self` and re-tag the state marker. Wire the module into `lib.rs` WITHOUT globbing it alongside the frozen `traits::*`, and add the `trybuild` dev-dependency that the Wave-3 compile-fail gate will consume.

This is the foundation that the UMAP/HDBSCAN shells (Plan 02), the compile-fail gate (Plan 03), and the PyO3 collapse (Plan 04) all build on. It is file-disjoint from every shell.

Implements D-03 (sealed `State` + `Unfit`/`Fitted` markers), D-05 (consuming `Fit` with `type Fitted`; `Predict`/`Transform` bound to fitted), D-06 (`PartialFit` consuming + multi-transition, defined-but-unused this phase), D-07 (COEXISTENCE — new module, `traits.rs` frozen). Adds the `trybuild` dev-dep per D-11.

Purpose: fix the single canonical typestate surface so the new estimators are born idiomatic and Phase 16 has one target shape to retrofit toward, with the 30-estimator green suite (Success Criterion 3 / BLDR-04) untouched.
Output: `typestate.rs`, the `lib.rs` registration, the `trybuild` dev-dep, and a runtime smoke test that the markers compose.
</objective>
