<objective>
Wrap all 12 `mlrs-algos` estimators as PyO3 `#[pyclass]` objects in `_mlrs`, driven by
the `any_estimator!` macro from Plan 02. Each wrapper carries its sklearn-named
hyperparameters, dispatches f32/f64 via its internal enum (D-06), releases the GIL
around the device trait calls (PY-03), materializes fitted attributes as host
buffers for the Python shim (D-03), and enforces the f64 capability guard (D-04).
Register all 12 on the module (`add_class`).

Purpose: This is the compiled half of PY-01/PY-02/PY-05. The pure-Python shim
(Plan 04) subclasses sklearn and delegates to these classes. Interface-first: the
wrappers expose a uniform low-level surface (fit / the relevant output method /
fitted-attr accessors) the shim calls.
Output: `estimators/{mod,linear,cluster,decomposition,neighbors}.rs`, the 12
pyclasses registered in `lib.rs`, and a Rust smoke test constructing each.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python call → device trait call | untrusted X/y + hyperparameters reach the device through the wrappers |
| unfitted estimator → accessor/predict | accessing fitted state before fit must be a clean error, not UB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | Tampering | Untrusted hyperparameter (k / alpha / eps / min_samples / n_components) → out-of-bounds device gather | mitigate | The `mlrs-algos` estimators validate hyperparameters in `fit` (InvalidK/InvalidAlpha/InvalidEps/...) BEFORE any launch (existing, unchanged); the wrapper surfaces the AlgoError as a `PyValueError` via Plan 02's error map. |
| T-06-09 | Information disclosure | Reading uninitialized fitted state from an unfitted wrapper | mitigate | The `Unfit` enum arm has no device buffers; accessors on `Unfit` return a clear not-fitted error the shim maps to `NotFittedError`. |
| T-06-10 | DoS | A Rust panic inside a `#[pymethods]` body unwinding into CPython | mitigate | PyO3 catches panics at the `#[pymethods]` boundary and converts them to a `PanicException`; the validated bridge + capability guard mean the expected error paths return `Err` (mapped PyErr), not panic. |

No unmitigated `high` threats. Untrusted-input handling reuses the existing
validated bridge (Plan 02) + the algos-level hyperparameter validation.
</threat_model>
